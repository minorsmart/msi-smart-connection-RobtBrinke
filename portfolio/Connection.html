Smart Connection

Bij Smart Connection is in de loop van de minor aandacht gegeven aan het maken en ontwerpen van websites. Alles wat in en rondom een website nodig en bezig is om een website werkend te hebben.

Codeertalen

Bij de lessen Smart Connection hebben we geleerd welke codeertalen er allemaal zijn en welke we gaan toepassen bij deze lessen. De normale technieken/talen die we gaan toepassen zijn:
-HTML
-CSS
-JS
Hierbij is HTML voor de normale opbouw van een website. Dus de tekst en de ruwe structuur van een website. CSS is daarentegen voor hoe een website er uit komt te zien. Het echt ontwerpen en vormgeven gebeurt hiermee. Met JS kunnen interactieve dingen toegevoegd worden. Zoals een weermeter of nieuwe coronabesmettingen zien.

API Toepassingen

Een voorbeeld van een API wat we hebben geleerd is bovenstaande Coronabesmettingen API. Deze API haalt de data op geeft deze data dan weer terug aan de code, daarna wordt de code laten zien op de site. Verder hebben we ook gewerkt om het weer te bekijken. Maar zo zou je ook bijvoorbeeld de tijd kunnen laten zien vanuit verschillende landen.

Zo kan ook Google Maps ingeladen worden op een site. Dit werkt via een API en hier kan dan een locatie laten zien van bijvoorbeeld waar een bedrijf zit, of waar je woont. Een API zoals google maps werkt heel snel en makkelijk. Deze code hoeft alleen op de website gezet te worden en dan hoeft alleen de locatie aangepast te worden naar wat je wilt. Deze locatie kan in de code aangepast worden. Zo heb ik de Ossenmarkt in Zwolle er neergezet, aangezien ik hier nu woon.

Ervaring

Het maken van een website met HTML en CSS was erg veel werk. Alleen dit veranderde erg snel want (gelukkig) wordt bij coderen veel gebruik gemaakt van google. Hiermee worden oplossingen opgezocht, kortere codeermanieren en formats om mee te werken. Hier heb ik dus ook zelf gebruik van gemaakt. Echter heb ik het zelf wel eerst geprobeerd om het zelf in elkaar te zetten. Wanneer ik dit alleen vergelijk met websites van anderen, die wel gebruik hebben gemaakt van andere sites, dan is die van mij erg kaal. Hierom heb ik gekozen om via Nicepage een website op te zetten. Waarbij ik de code vervolgens via Visual Studio Code gewoon kan veranderen. Dat is uiteindelijk deze website geworden. 

Programmeren in Business

Het werken met programmeren van websites in een business aspect kan op verschillende manieren toegepast worden. Hieronder heb ik een paar voorbeelden beschreven hoe met programmeren om gegaan kan worden, via een business perspectief.

Beveiliging
Netwerk beveiliging of web security is een steeds belangrijker aspect bij websites. Dit is natuurlijk ook te zien aan de recente hack van de HAN.
Het hacken van iemand kan op verschillende manieren. Over het internet gaat het vooral op drie verschillende manieren. Cross-site scripting, dit is wanneer een applicatie aangepast wordt door een hacker zodat de hacker informatie verkrijgt van de bezoeker, dit gebeurt wanneer er bijvoorbeeld een fout in de code zit [4]. Hackers kunnen veel met gevoelige informatie, en dan vooral met credit en debit card gegevens. Deze manier van hacken heet sensitive data theft. De derde manier is Cross-site request forgery, deze manier zorgt ervoor dat de bezoeker geforceerd wordt om data te sturen naar een derde partij [3]. 
51% van kleine bedrijven zijn onder de indruk dat veiligheid van hun systemen of website niet veel uit maakt. Wie zou hun willen hacken en wat schieten de hackers ermee op [10].
Wanneer deze kleine bedrijven een relatief goedkoop pakket zouden kunnen kopen om de producten en diensten veilig te houden zou dit dus aanzienlijke sales kunnen opleveren. Mits het goed overgebracht wordt.

Big data
Het verzamelen van data wordt steeds meer aandacht aan gegeven. Deze data wordt steeds uitgebreider en specifieker door de zoekgeschiedenis en gedrag van de gebruikers. Met deze data kan verschillende dingen aangetoond worden, zoals:
   -Het leren kennen van je klanten.
   -Organisaties laten weten hoe ze loyaliteit kunnen opbouwen en trends kunnen               volgen.
   -Organisaties kunnen op basis van persoonlijkheid iemand benaderen
   -Big data zorgt voor meer kennis en begrip over de klant en het gedrag van de klant
   -Betere communicatie met de klant [1][2][5].

Naast marketing en klanten kennis is big data ook toepasbaar om productontwikkeling te stimuleren en betere besluitvorming. Big data kan gebruikt worden om met de concurrentie mee te gaan. Met veel data kunnen organisaties verschillende processen implementeren om het gedrag van een klant te volgen, bepalen wat succesvolle producten zijn en waar concurrenten mee bezig zijn [6].

De term big data lijkt na al deze punten erg mooi en te goed om waar te zijn. Echter zijn veel bedrijven die big data kennen en willen invoeren alleen niet weten waar ze moeten beginnen. Veel bedrijven willen big data invoeren zonder dat deze enig idee hebben waarvoor. Dat is de eerste stap, iets hebben waarvoor het dient, een doel voor ogen hebben. Verder moet het data verzamelen natuurlijk geen extra werk voor het hele bedrijf opleveren of geen doel voor de klant hebben [7][8].
Data kan snel, veel, verzameld worden, hierom is het belangrijk dat de goede tools aanwezig zijn om hiermee om te gaan. Big data tools zoals Apache Hadoop, Tableau, Power Bi en QlikView kunnen hiervoor gebruikt worden. Het is dus niet nodig om zelf een tool te laten ontwikkelen [8][9].
